Resolução de Problemas usando Ordenação

A ordenação é uma operação fundamental em Ciência da Computação. Em muitos casos, ordenar os dados pode simplificar a resolução de problemas aparentemente mais difíceis. Nesta aula serão apresentados os conceitos principais sobre ordenação e como ela pode ser usada na resolução de problemas.

Existem muitos algoritmos de ordenação que variam bastante em relação ao seu funcionamento e desempenho. Entretanto, neste material não serão abordados algoritmos específicos de ordenação, mas sim o conceito geral de ordenação e como ele pode ser aplicado na resolução de problemas.

Importante!

Os exemplos usados neste material estão em C++. Para que a discussão entre algoritmo e código fique alinhada, vamos considerar que os vetores com \(n\) elementos vão de \(0\) a \(n-1\).

Definição 1 (Ordenação) Dado um vetor \(V\) com \(n\) elementos quaisquer, \(V = [ v_0, v_1, v_2, \dots, v_{n-1} ]\), uma ordenação de \(V\) permuta seus elementos tal que o vetor ordenado \(V' = [v_0', v_1', v_2', \dots, v_{n-1}']\) respeite a restrição \(v_0' \leq v_1' \leq v_2' \leq \dots \leq v_{n-1}'\), onde \(\leq\) é a relação a qual queremos impor à ordem dos elementos.

Exemplo – Vetor de Inteiros

Dado o vetor de inteiros \(V = [4, 2, 7, 1, 3]\), uma ordenação crescente de \(V\) resulta no vetor \(V' = [1, 2, 3, 4, 7]\). Note que os elementos em \(V\) são os mesmos que em \(V'\), apenas em uma ordem diferente. Também é relevante ressaltar que a ordenação crescente é consequência da relação \(a \leq b\) entre dois inteiros quaisquer \(a\) e \(b\).

Exemplo – Vetor de Strings

Dado o vetor de strings \(S = [\text{``banana''}, \text{``abacaxi"}, \text{``laranja"}]\), uma ordenação lexicográfica (ordem alfabética) resulta no vetor \(S' = [\text{``abacaxi"}, \text{``banana"}, \text{``laranja"}]\). A relação usada para ordenar as strings é a ordem lexicográfica, que compara os caracteres das strings da esquerda para a direita em ordem alfabética.

Propriedades Fundamentais de Vetores Ordenados

A partir da definição de ordenação, podemos deduzir algumas propriedades importantes dos vetores ordenados.

Propriedade 1 Um vetor vazio está sempre ordenado.

Propriedade 2 Um vetor unitário está sempre ordenado.

Propriedade 3 Um elemento em uma posição \(i\) qualquer de um vetor ordenado \(V\) é maior ou igual a todo elemento no subvetor \(V[0 \dots i-1].\)

Propriedade 4 Um elemento em uma posição \(i\) qualquer de um vetor ordenado \(V\) é menor ou igual a todo elemento no subvetor \(V[i+1 \dots n-1].\)

Resolução de Problemas

Ordenar um conjunto de dados não é apenas uma questão de organização. A estrutura imposta pela ordenação pode ser explorada para resolver problemas de forma mais eficiente. A seguir vamos explorar alguns problemas cuja solução usando ordenação é mais eficiente do que uma abordagem força-bruta.

Exemplo 1: Verificar a Existência de Elementos Duplicados em um Vetor

Dado um vetor V com n elementos quaisquer, verifique se há elementos duplicados no vetor. Por exemplo, para o vetor V = [1, 2, 3, 4, 5], a resposta é “não” (false), enquanto para o vetor V = [1, 2, 3, 4, 5, 3], a resposta é “sim” (true), uma vez que o elemento 3 aparece duas vezes no vetor. Os elementos do vetor podem ser permutaods durante a verificação.

Resolução por Força-Bruta

Uma solução força-bruta consiste em testar todas as possibilidades. Neste caso, envolve comparar todos os pares de elementos do vetor, verificando se algum par é igual. Segue o algoritmo implementado em C++.

template<typename T>
bool temDuplicadosFB(vector<T> &V) {
  int n = V.size();
  for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
      if (V[i] == V[j]) {
        return true;
      }
    }
  }
  return false;
}

Como há \(\frac{n^2+n}{2}\) pares possíveis, no pior caso (quando apenas os elementos do ultimo par forem iguais) o custo deste algoritmo é proporcional ao quadrado do número de elementos do vetor! Ou seja, se o vetor for muito grande, este algoritmo vai demorar muito pra terminar.

Resolução Usando Ordenação

O que acontece se ordenarmos o vetor? Sabemos, pela Def. 1 (Ordenação), que o vetor assume a ordem imposta. Como consequência, os elementos que são iguais ocuparão posições adjacentes! Com isso, o problema de verificação de duplicadas se reduz a verificar se há elementos adjacentes iguais! Segue o algoritmo implementado em C++.

template<typename T>
bool temDuplicadosOrd(vector<T> &V) {
    sort(V.begin(), V.end()); // Ordena o vetor
    int n = V.size();
    for (int i = 0; i < n - 1; i++) {
        if (V[i] == V[i + 1]) { // Verifica elementos adjacentes
            return true;
        }
    }
    return false;
}

A ordenação do vetor pode ser feita em tempo \(n \log n\) (usando um algoritmo de ordenação eficiente), e a verificação de elementos adjacentes em tempo \(n\). Portanto, o custo total deste algoritmo é proporcional a \(n \log n\), que é significativamente melhor do que o custo \(n^2\) do algoritmo força-bruta.

Exemplo 2: K-ésimo Maior

Dado um vetor \(V\) com \(n\) elementos inteiros positivos quaisquer e sem repetições, retorne o k-ésimo maior elemento do vetor. Por exemplo, se \(k=1\), deve-se retornar o maior elemento do vetor, com \(k=2\) retorna-se o segundo maior elemento do vetor, e assim por diante. Caso o vetor tenha menos que \(k\) elementos, retorne -1. Os elementos do vetor podem ser permutados durante a busca.

Resolução por Força-Bruta

Uma solução por força-bruta para este problema consiste em encontrar primeiramente o maior elemento entre as posições 0 e n-1. Considere que este primeiro elemento foi encontrado na posição p. Ele então é trocado com a posição n-1. Se k==1, retorna-se v[n-1]. Caso contrário, a posição do próximo maior elemento é encontrada (posição p) e seu valor é trocado com a posição V[n-2], que é retornado caso k==2. Este algoritmo é repetido k vezes. A implementação em C++ é mostrada a seguir.

int kesimoMaior(std::vector<int> &V, int k){
    int n = V.size();
    if(k > n) return -1;
    for(int i = 1; i <= k; i++){
        int maior = 0;
        for(int j = 1; j <= (n-i); j++){
            if(V[j] > V[maior]){
                maior = j;
            }
        }
        std::swap(V[n-i], V[maior]);
        if(k == i)
            return V[n-i];
    }
    return -1;
}

O custo deste algoritmo é proporcional a \(k \cdot n\). Portanto, se \(k\) for grande suficiente (próximo de n), este algoritmo tem custo quadrático, o que é muito ineficiente para \(n\) grande.

Resolução Usando Ordenação

O que acontece se ordenarmos o vetor? Sabemos, pela Def. 1 (Ordenação), que o vetor assume a ordem imposta. Portanto, \(v_0' \leq v_1' \leq v_2' \leq \dots \leq v_{n-1}'\). Com isto, podemos inferir que, após a ordenação, o maior elemento do vetor estará na posição n-1, o segundo maior na posição n-2, e assim por diante! Com isto, o problema de encontrar o k-ésimo maior é simplificado para o acesso direto à posição do qual o k-ésimo elemento deve ocupar. A implementação em C++ é mostrada a seguir.

int kesimoMaiorOrd(std::vector<int> &V, int k){
    int n = V.size();
    if(k > n) return -1;
    std::sort(V.begin(), V.end()); // Ordena o vetor
    return V[n-k]; // Retorna o k-ésimo maior
}

O custo deste algoritmo é o custo de ordenar o vetor. Usando um algoritmo de ordenação eficiente, é possível obter o vetor ordenado em \(n \lg n\). Portanto, o custo total deste algoritmo é proporcional a \(n \log n\), que é significativamente melhor do que o custo \(n^2\) do algoritmo força-bruta.

Exercícios

1. Agora considere que os elementos de V não podem ser permutados na resolução do Exemplo 1. Altere ambas soluções para que agora respeitem esta restrição, com o mínimo de mudanças no código. Você pode usar espaço (memória) extra.

2. Altere a solução eficiente do Exemplo 1 para que a função retorne um novo vetor com os mesmos valores, mas sem repetições. Considere também que os elementos do V não podem ser permutados. A função deve ter a assinatura vector<T> semDuplicatas(const vector<T> &V).

3. Altere a solução eficiente do Exemplo 2 de forma que possam haver repetições no vetor. Também considere que os elementos de V não podem ser permutados. Você pode usar espaço extra proporcional ao tamanho do vetor V. Caso não haja um k-ésimo maior elemento, retorne -1. Considere tudo que você aprendeu até aqui para resolver este problema. A função deve ter a assinatura vector<int> kesimoMaiorOrdReps(const std::vector<int> &V, int k).
